# SPTNR Documentation - Screenshots Guide

This directory contains screenshots referenced in the documentation.

## Taking Screenshots

When adding screenshots to the documentation:

1. **Naming Convention**: Use descriptive names matching the documentation references
   - Example: `dashboard_overview.png`, `artist_detail.png`

2. **Recommended Dimensions**: 
   - Full page: 1920x1080 or 1440x900
   - Partial views: 800-1200px width
   - UI elements: As needed, but maintain clarity

3. **Format**: PNG preferred for UI screenshots (crisp text)

4. **Content**:
   - Show actual SPTNR interface
   - Include relevant UI elements and features
   - Use realistic sample data when possible
   - Highlight important features with arrows/boxes if helpful

## How to Reference in Documentation

In markdown files, reference screenshots like this:

```markdown
![Description](screenshots/filename.png)
```

Or with a link:
```markdown
### Feature Name
![Feature Screenshot](screenshots/feature_name.png)

Description of the feature...
```
